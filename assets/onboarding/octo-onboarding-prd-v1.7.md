## 1. 需求背景

### 1.1 产品概述

Octo 是一款 AI 协作平台，提供 **SaaS 版本**和**本地化部署版本**两种形态。三类用户角色在首次使用时面临不同的 Onboarding 挑战。

> **三个角色定义：**
> - **Admin 管理员** — 管理后台超级管理员，负责系统初始化配置。仅存在于本地化部署版（SaaS 版的 Admin 是我们自己）
> - **Space 管理员** — Space 拥有者/管理员，负责管理自己的工作空间。两版本共有
> - **普通用户** — 通过邀请码/链接注册加入 Space 的成员。两版本共有

### 1.2 当前痛点

| 角色 | 当前痛点 | 影响 |
|------|---------|------|
| **Admin 管理员** | 需跨 6 个页面手动配置（登录→创建Space→配SMTP→开邮箱注册→生成邀请码→复制分享），SMTP 概念难理解 | 配置耗时 15-30 分钟，易中途放弃 |
| **Space 管理员** | 首次进入 Space 后不知道在哪管理成员/邀请码，需自行摸索空间管理功能 | 无法高效完成团队搭建 |
| **普通用户** | 注册后页面空空如也，没有会话和群聊，不知道该干什么 | 首次体验差，留存风险高 |

### 1.3 优化目标

| 指标 | 当前 | 目标 |
|------|------|------|
| Admin 操作步骤 | 6 步 | 3 步 |
| Admin 配置时间 | 15-30 分钟 | 5 分钟 |
| Space 管理员引导 | 无 | 3+5 步 Walk-through |
| 用户首次体验 | 空白页面 | 自动加入全员群 + 产品介绍弹窗 + Bot 引导 |

## 3. 功能详述 — Admin 管理员视角

> ℹ️ 本章节所有功能**仅适用于本地化部署版本**。SaaS 版的 Admin 是平台运维团队，不需要引导。

### #1 首次登录 Setup Wizard 「P0」

**功能描述：** 管理员首次登录管理后台时，触发**阻断式引导向导**，通过 3 个步骤完成系统初始化配置，替代原有的 6 步手动配置流程。

**触发条件：**
- 管理员登录管理后台后，系统检测到**尚未创建任何 Space** → 自动进入 Setup Wizard
- 已创建过 Space → 直接进入管理后台首页（不触发向导）

**流程步骤：** 🔐 登录 → 🏢 创建空间 → 📧 配置邮件 → ✅ 完成页

**Step 1: 管理员登录**
- 输入账号和密码（由部署脚本提供）
- 登录验证通过后自动进入 Step 2

**Step 2: 创建工作空间**

| 字段 | 必填 | 说明 |
|------|------|------|
| 空间名称 | ✅ 是 | Space 的显示名称，同时作为全员群名称 |
| Logo URL | 否 | 可选，Space 图标 |
| 人数上限 | 否 | 可选，默认不限制 |
| 加入方式 | 否 | 「直接加入」（默认）或「需审批」 |

**在Setup Wizard创建Space：** Space 拥有者 = Admin 本人（自动绑定）

**Step 3: 配置邮件服务**

| 字段 | 必填 | 说明 |
|------|------|------|
| 发件邮箱 | ✅ 是 | 技术支持邮箱（发件人地址） |
| SMTP 服务器 | ✅ 是 | 格式 host:port（支持自动识别，见 #6） |
| SMTP 授权码 | ✅ 是 | 邮箱的授权码（非登录密码），有帮助弹窗（见 #7） |

**发送测试邮件：** 三个必填字段填写完成后，提供「📧 发送测试邮件」按钮，点击后向发件邮箱发送一封测试邮件，验证 SMTP 配置是否正确。该功能复用管理后台已有的发送测试邮件能力。测试结果实时反馈：成功显示 ✅ 提示，失败显示 ❌ 错误信息。测试邮件为可选操作，不阻塞「下一步」。

**完成页：**
- 显示创建成功提示
- 展示**邀请码**和**邀请链接**，支持一键复制
- 点击「进入管理后台」按钮 → 跳转管理后台首页

**交互细节：**
- 阻断式：向导期间无法跳过或关闭，必须完成所有步骤
- 步骤间支持「← 上一步」返回（文字链接样式，左侧弱化）
- 「下一步」按钮颜色：#7171EE（品牌紫色）
- 表单实时校验，未填必填项时「下一步」按钮置灰

### #2 邮箱注册默认开启 「P0」

**功能描述：** 本地化部署版本中，邮箱注册功能**默认开启**（当前默认关闭）。管理员完成 SMTP 配置后，用户即可通过邮箱注册加入 Space。

**变更点：**
- 系统部署后邮箱注册默认为「开启」状态
- 管理员可在管理后台手动关闭

### #9 未配置拦截提示 「P0」

**功能描述：** 本地化部署版中，系统尚未完成初始配置时（未创建 Space 或未配置 SMTP），用户在 Octo 登录页点击「登录」或「注册」→ 弹出阻断式提示弹窗。

**触发操作：** 用户在 Octo 登录页点击以下任一操作时弹出：
- 「登录」按钮
- 「注册」按钮
- 「扫码登录」
- 「忘记密码」

**弹窗内容：**
- 标题：「⚠️ 系统尚未配置」
- 提示文案：列出未完成的配置项（未创建工作空间 / 未配置邮件服务）
- 引导文案：「请联系管理员完成配置」
- 关闭按钮：「我知道了」

**样式：**
- 位置：登录页底部居中

### #6 SMTP 自动识别 + 快捷选择 「P1」

**功能描述：** 在 Setup Wizard Step 3 中，当管理员输入发件邮箱后，系统根据邮箱后缀**自动推测并填充** SMTP 服务器地址。

**自动识别规则：**

| 邮箱后缀 | 自动填充 SMTP 服务器 |
|---------|-------------------|
| @qq.com | smtp.qq.com:465 |
| @163.com | smtp.163.com:465 |
| @gmail.com | smtp.gmail.com:587 |
| @outlook.com / @hotmail.com | smtp.office365.com:587 |
| 其他 | 不自动填充，用户手动输入 |

**快捷标签：** 在 SMTP 服务器输入框上方提供常见服务商快捷标签按钮（QQ邮箱 / 163 / Gmail / Outlook），点击一键填充服务器地址。

### #11 管理后台 Walk-through 引导 「P1」

**功能描述：** Setup Wizard 完成后进入管理后台，触发**非阻断式 4 步引导**，帮助管理员了解后台核心功能区域。

**引导步骤：**

| 步骤 | 高亮区域 | 引导说明 |
|------|---------|---------|
| ① | Space 管理表格 | **标题：**「🏢 你的 Space 已创建！」**描述：** 这里是 Space 管理页面。你刚才在引导中创建的 Space 已经出现在列表中。你可以点击右上角「＋ 创建空间」来新增更多 Space。 |
| ② | 操作列「邀请码」按钮 | **标题：**「🔑 邀请码入口」**描述：** 每个 Space 操作列中都有「邀请码」按钮，点击即可打开该 Space 的详情面板，查看和复制邀请码。用户点击该按钮后，右侧滑出 Space 详情侧边栏 |
| ③ | 侧边栏中的邀请码区域（tab 栏 + 邀请码列表 + 生成按钮） | **标题：**「📋 复制邀请码」**描述：** 这里是邀请码管理区域。包含当前有效的邀请码列表，可以直接复制。也可以点击「+ 生成邀请码」创建新的邀请码。 |
| ④ | 左侧「应用 Bot」导航菜单 | **标题：**「🤖 应用 Bot 管理」**描述：** 在这里可以配置和管理 AI Bot 及第三方应用。配置后的 Bot 对所有 Space 可见，成员可在 Octo 中直接使用。 |

> ⚠️ **第②→③步交互联动：** 第②步高亮「邀请码」按钮，用户点击后右侧滑出 Space 详情侧边栏，引导自动推进到第③步，高亮侧边栏内的邀请码区域。

**交互规则：**
- 非阻断式：每步可点「跳过」或「下一步」
- 聚光灯效果：半透明遮罩 + 目标区域高亮
- 仅首次进入管理后台时触发

## 4. 功能详述 — Space 管理员视角

> ℹ️ 本章节功能**两版本共有**（SaaS + 本地化部署），除非特别标注版本差异。

### #3 创建 Space 时自动创建全员群 「P0」

**功能描述：** 创建 Space 完成时，系统自动创建一个**全员群**，Space 拥有者为群主，群名跟随 Space 名称。后续新用户加入 Space 时自动成为全员群成员。

**后端逻辑：**
- 创建 Space 完成 → 自动创建群聊，群主 = Space 拥有者
- 群名称 = Space名称 + 全员群（如「明略科技全员群」）
- 新用户通过邀请码加入 Space → 自动加入全员群
- 系统在群内发送提示消息：「Admin 管理员创建了 {Space名称} 全员群」
- 普通用户加入时额外显示：「{用户名} 加入了群聊」

**前端表现 — 消息列表预置：**

所有用户（管理员和普通用户）首次进入 Space 时，「最近」tab 消息列表中预置全员群聊天项（含未读 badge）。

> 🔀 **版本差异 — 未读 badge 数量：**
> - **SaaS 版**：「最近」tab 显示 **3** 条未读（全员群 + 向导虾 + BotFather）
> - **本地化部署版**：「最近」tab 显示 **2** 条未读（全员群 + BotFather，无向导虾）

**消息列表项排序：**

| 位置 | SaaS 版 | 本地化部署版 |
|------|---------|------------|
| 第 1 项 | 明略科技全员群 (未读 badge) | 明略科技全员群 (未读 badge) |
| 第 2 项 | 向导虾 (未读 badge) | BotFather (未读 badge) |
| 第 3 项 | BotFather (未读 badge) | — |

### #4 Space 拥有者免邀请码自动加入 「P0」

**功能描述：** 被指定为 Space 拥有者的用户，**无需填写邀请码**，Space 创建完成后自动加入对应 Space。

**触发场景：**
- **本地化部署**：Admin 在 Setup Wizard Step 2 创建 Space 时，Space 拥有者 = Admin 本人（自动绑定），后续Admin管理员在管理后台创建 Space 并指定 Space 拥有者 → 该用户自动加入
- **SaaS 版**：平台运营创建 Space 并指定 Space 拥有者 → 该用户自动加入

### #10 首次进入 Walk-through 引导（3+5 步） 「P1」

**功能描述：** Space 管理员首次进入 Space 时触发**两阶段非阻断式引导**，帮助管理员快速了解核心功能入口。两个阶段在各自页面首次进入时独立触发。

**阶段一：Octo 主页面（3 步）**

| 步骤 | 高亮区域 | 引导说明 |
|------|---------|---------|
| ① | 全屏（无高亮） | **标题：**「👋 欢迎来到你的工作空间！」**描述：** 作为 Space 管理员，你可以管理成员、生成邀请码、配置 Bot 等。让我们先看看如何进入管理页面。 |
| ② | 左下角 ⚙️ 齿轮图标 | **标题：**「⚙️ 点击设置按钮」**描述：** 点击左下角的齿轮图标，打开设置菜单。 |
| ③ | 设置菜单中「🏢 空间管理」选项 | **标题：**「📋 空间管理」**描述：** 点击「空间管理」将打开独立的管理页面，你可以在那里管理成员、生成邀请码、审批加入申请、配置应用 Bot 等。 |

**阶段一结束后 → 在当前 Octo 主页面弹出 #12 产品介绍弹窗**，完成后续引导流程。

> ⚠️ **注意区分两个阶段的独立性：** 阶段一 + 欢迎弹窗在 Octo 主页面完成；阶段二在空间管理页面独立触发（Space 管理员第一次进入对应 Space 的空间管理页面时），与阶段一和欢迎弹窗是分开的流程。

**阶段二：空间管理页面（5 步）**

**触发时机：** Space 管理员**第一次进入**对应 Space 的空间管理页面时触发，与阶段一独立（不依赖阶段一是否完成）。无论用户是从 Octo 主页面点击设置中的「空间管理」打开，还是直接输入空间管理页面的 URL 打开，均会触发引导。

| 步骤 | 高亮区域 | 引导说明 |
|------|---------|---------|
| ① | 空间信息卡片 | **标题：**「🏢 空间基本信息」**描述：** 这里展示 Space 的名称、加入方式（直接加入/需审批）和当前成员数量。管理员可以在此确认配置。 |
| ② | 成员 tab | **标题：**「👥 成员管理」**描述：** 「成员」标签页显示所有已加入的成员，可以查看角色、状态，并进行移除操作。管理员角色会以橙色标签高亮。 |
| ③ | 邀请码 tab | **标题：**「🔑 邀请码管理」**描述：** 在「邀请码」标签页可以生成新的邀请码，分享给需要加入的成员。每个邀请码可设置有效期和使用次数。 |
| ④ | 加入申请 tab | **标题：**「📝 加入申请」**描述：** 当 Space 加入方式设为「需审批」时，用户的加入申请会出现在这里。管理员可以在此批准或拒绝申请。 |
| ⑤ | Bot tab | **标题：**「🤖 应用 Bot 管理」**描述：** 在「应用 Bot」标签页可以查看和管理 Space 中接入的 AI Bot，包括启用/禁用和权限配置。 |

**交互规则：**
- 非阻断式：每步支持「跳过引导」和「下一步」
- 聚光灯效果：目标区域高亮 + 半透明遮罩 + 引导气泡
- 引导气泡智能定位：target 在屏幕下半部分时，气泡显示在 target 上方
- 仅首次进入空间管理页面触发，后续进入不再显示

## 5. 功能详述 — 普通用户视角

> ℹ️ 本章节功能面向**所有用户**（包括 Space 管理员和普通用户），部分功能有版本差异和角色差异，已逐一标注。

### #12 Octo 产品介绍弹窗 「P0」〔两版本共有〕

**功能描述：** 用户首次进入 Space 时自动弹出全屏 Octo 产品介绍弹窗。

**触发时机（角色差异）：**

| 角色 | 触发时机 | 弹出页面 |
|------|---------|---------|
| **Space 管理员** | Walk-through **阶段一**结束后（点击「跳过」或完成第③步）自动弹出 | Octo 主页面 |
| **普通用户** | 进入 Space 后**直接弹出**（无 Walk-through） | Octo 主页面 |
