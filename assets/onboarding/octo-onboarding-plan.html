🔍 问题背景

Octo 平台包含 SaaS 版本和本地化部署版本两种形态。不同角色在 Onboarding 中面临不同的痛点：

ℹ️ 三个角色定义：
· Admin 管理员：管理后台超级管理员，负责系统初始化配置（仅本地化部署版，SaaS 版的 Admin 是我们自己）
· Space 管理员：Space 拥有者/管理员，负责管理自己的工作空间（两版本共有）
· 普通用户：通过邀请码/链接注册加入 Space 的成员（两版本共有）

Admin 管理员需跨 6 个页面手动配置，不知道下一步做什么，SMTP 概念不理解

Admin 视角 Feature 清单

  • P0 #1 首次登录 Setup Wizard
    管理员首次登录触发阻断式向导，3步完成配置。条件触发逻辑：没创建过Space → 进引导；已创建过 → 直接进管理后台首页。
  • P0 #2 邮箱注册默认开启
    本地化部署版默认开启邮箱注册功能，管理员有需要时再手动关闭。
  • P0 #9 未配置拦截提示
    本地化部署版未完成初始配置时，用户在 Octo 登录页点击「登录」或「注册」→ 弹出「系统尚未配置」提示，列出未完成项，引导前往管理后台。
  • P1 #5 Octo 登录页管理后台入口
    登录页底部弱化文字链接「⚙️ 管理后台」，点击新开标签页打开管理后台。
  • P1 #6 SMTP 自动识别 + 快捷选择
    根据邮箱后缀自动推测 SMTP 服务器；提供 QQ/163/Gmail 等快捷标签按钮一键填充。
  • P1 #7 SMTP 授权码帮助文档
    内嵌帮助弹窗「如何获取授权码？」，包含主流邮箱的分步教程。
  • P1 #11 管理后台 Walk-through 引导
    Setup Wizard 完成后进入管理后台，触发 4 步引导：
    ① 高亮 Space 管理表格，告知已创建的 Space 并引导新增；
    ② 高亮操作列「邀请码」按钮，告知可以在此查看邀请码；
    ③ 打开 Space 详情侧边栏，高亮邀请码 tab 区域（tab栏 + 邀请码列表 + 生成按钮），告知可随时复制或生成新邀请码；
    ④ 高亮左侧「应用 Bot」菜单，说明可配置 Bot 到所有 Space 下可见。
  • P2 #8 部署脚本终端输出优化
    部署完成后终端直接输出管理后台地址 + 初始账号密码，一屏展示关键信息。

👑 视角二：Space 管理员 两版本共有

🏢
Space 管理员（Space 拥有者）

创建 Space 时被指定的拥有者，拥有 Space 管理权。登录 Octo 后管理自己的空间。SaaS 和本地化部署均适用。

Walk-through 引导流程（分两阶段）

Space 管理员的引导拆分为两个独立页面，分别触发：

① Octo 主页面引导（3 步）

首次进入 Space 时触发，引导找到空间管理入口：

  1. 欢迎提示 — 欢迎来到你的空间！让我们了解如何管理。
  2. 设置按钮 — 高亮左下角⚙️齿轮图标，引导点击打开设置菜单。
  3. 空间管理入口 — 高亮设置菜单中「🏢 空间管理」选项，点击后将打开独立的空间管理页面。

② 空间管理页面引导（5 步）

管理员首次打开空间管理页面（独立网页）时触发，介绍各功能区：

  1. 空间基本信息 — 展示 Space 名称、加入方式和当前成员数。
  2. 成员管理 — 查看和管理所有空间成员，支持搜索、角色管理和移除操作。
  3. 邀请码管理 — 生成和管理邀请码/邀请链接，分享给新成员。
  4. 加入申请 — 当加入方式设为「需审批」时，用户的申请在此批准或拒绝。
  5. 应用 Bot — 管理接入到空间的 AI Bot 和第三方应用。

Space 管理员视角 Feature 清单

  • P0 #3 创建 Space 时自动创建全员群
    创建 Space 完成时，自动以 Space 拥有者为群主创建全员群（群名跟随 Space 名称）。新用户加入时自动成为群成员。

    消息列表预置：所有用户（管理员和普通用户）首次进入 Space 时，「最近」tab 消息列表中预置全员群聊天项（含未读 badge），点击进入可看到系统提示「Admin 管理员创建了 XX 全员群」。普通用户还会看到「用户名 加入了群聊」的提示。

    未读 badge 数量：SaaS 版「最近」tab 显示 3 条未读（全员群 + 向导虾 + BotFather）；本地化部署版显示 2 条未读（全员群 + BotFather，无向导虾）。
  • P0 #4 Space 拥有者免邀请码自动加入
    被指定为 Space 拥有者的用户无需填写邀请码，创建后自动加入对应 Space。
  • P1 #10 首次进入 Walk-through 引导（3+5 步）
    Space 管理员首次进入 Space 时触发两阶段引导：① Octo 主页面 3 步（欢迎→设置按钮→空间管理入口）；② 空间管理独立页面 5 步（基本信息→成员→邀请码→加入申请→Bot）。两个页面分别在首次进入时独立触发，每步可跳过。

    帮助菜单角色差异：Space 管理员点击左下角 ❓ 帮助按钮时，菜单中额外显示「Space 管理后台」入口（点击跳转空间管理页）；普通用户的帮助菜单无此选项。
💡 Walk-through 引导为非阻断式（可跳过）。两个阶段分别在对应页面首次打开时独立触发（Octo 主页面 3 步 + 空间管理页 5 步）。空间管理是独立网页，从 Octo 主页面点击「空间管理」后打开。

首次进入体验优化

  • P0 #3 自动创建全员群
    创建 Space 时自动创建全员群，Space 拥有者为群主，群名跟随 Space 名称。进入 Octo 时不再是空白页面，立即看到全员群和群聊消息，有即时归属感。
  • P0 #16 普通用户自动加入全员群
    普通用户通过邀请码注册并加入 Space 后，系统自动将其加入全员群，无需手动操作。全员群内显示「{用户名} 加入了群聊」系统提示，帮助团队了解新成员动态。
  • P0 #12 Octo 产品介绍弹窗
    首次进入 Space 时自动弹出 Octo 产品介绍视频弹窗。触发时机因角色而异：Space 管理员在 Walk-through 引导完成后弹出；普通用户进入 Space 后直接弹出（无 Walk-through）。弹窗包含产品介绍视频 + 导向按钮：SaaS 版为「找向导虾聊聊」，本地化部署版为「查看产品功能文档」。

    关闭弹窗后的引导提示：用户关闭弹窗（点击导向按钮或「跳过」）后，左下角 ❓ 帮助按钮旁自动弹出一个轻量 tooltip「随时点这里查看新手引导 👋」，10 秒后自动消失。帮助用户知道后续可以随时唤起引导入口。
  • P1 #13 BotFather 私聊引导
    消息列表中预置 BotFather 私聊消息，通过对话式交互告知用户基本操作步骤（如何发消息、创建群聊、添加 Bot 等），提供交互式的新手教程。所有用户（管理员和普通用户）首次进入 Space 时都会看到。
  • P1 #14 「向导虾」产品向导 Bot（SaaS 版）
    SaaS 版本中，消息列表额外预置「向导虾」官方向导 Bot，作为 BotFather 之外的产品功能向导。通过对话式交互带领用户熟悉 Octo 核心功能（发消息、群聊、Bot、任务等）。

    角色差异：Space 管理员的向导虾消息中额外包含「⚙️ 空间管理后台」入口链接和「邀请同事加入」引导；普通用户的向导虾不包含管理后台和邀请相关内容。
  • P1 #15 新手引导帮助菜单（❓ 按钮）
    左下角 ❓ 帮助按钮，点击弹出「你想先做什么？」新手引导菜单。菜单项因版本和角色而异：

    SaaS 版 · Space 管理员：跟向导虾聊聊 / 领养龙虾（下载客户端） / 绑定龙虾 / Space 管理后台
    SaaS 版 · 普通用户：跟向导虾聊聊 / 领养龙虾 / 绑定龙虾
    本地化部署版 · Space 管理员：领养龙虾 / 绑定龙虾 / 打开 Octo 使用手册 / Space 管理后台
    本地化部署版 · 普通用户：领养龙虾 / 绑定龙虾 / 打开 Octo 使用手册

    用户可随时通过 ❓ 按钮重新打开菜单，作为长期可用的引导入口。
💡 版本差异 & 角色差异：
· 普通用户：进入 Space → 直接弹产品介绍弹窗 → 关闭后左下角弹新手引导提示 → 消息列表有全员群 + BotFather + 向导虾(SaaS)
· Space 管理员：进入 Space → Walk-through 引导 → 引导完弹产品介绍弹窗 → 关闭后左下角弹新手引导提示 → 消息列表同上
· SaaS 版：弹窗按钮「找向导虾聊聊」，消息列表有「向导虾」官方 Bot（管理员版多含空间管理入口 + 邀请引导）
· 本地化部署版：弹窗按钮「查看产品功能文档」，无向导虾
· 帮助菜单：Space 管理员多一个「Space 管理后台」入口（SaaS 和本地化部署版均有），普通用户无此选项
⚠️ 「需审批」场景：用户提交加入申请后，页面提示「等待管理员审批」。Space 管理员在空间管理 → 加入申请 中处理。

🗓️ 实施路线

第一阶段（P0 · 核心流程打通）
Admin 视角：Setup Wizard + 邮箱注册默认开启 + 未配置拦截提示
Space 管理员视角：创建 Space 自动创建全员群 + 拥有者免邀请码加入 + Octo 产品介绍弹窗
用户视角：新成员自动加入全员群(#16) + Octo 产品介绍弹窗（直接弹出，无 Walk-through）
目标：三个角色的核心路径 5 分钟内走通
第二阶段（P1 · 体验打磨）
Admin 视角：登录页管理后台入口 + SMTP 智能识别 + 帮助文档 + 管理后台 Walk-through 引导
Space 管理员视角：Walk-through 引导（8步聚光灯）
用户视角：BotFather 私聊引导（两版本共有）/ 「向导虾」产品向导 Bot（SaaS 额外）/ 新手引导帮助菜单（❓ 按钮，两版本共有）
目标：消除认知门槛，降低学习成本
第三阶段（P2 · 锦上添花）
Admin 视角：部署脚本终端输出优化
长远考虑：内置默认发件服务（零邮箱配置可用）
目标：部署即用，零配置上手
